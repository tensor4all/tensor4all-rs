## Issues Found

1. **README/getting-started dependency instructions fail for a fresh Cargo user.** The documented `tensor4all-simplett = "0.1"` dependency is not available in the crates.io index, so the first setup path fails before compiling.
2. **Source-checkout usage is not explicit in the quick start.** Current APIs work through local path dependencies, but users have to infer that workaround.
3. **C API docs are more design-oriented than onboarding-oriented.** The API works, including last-error retrieval, but a minimal external C/binding smoke test would make setup easier.

## Suggestions

1. Update README and `docs/book/src/getting-started.md` to either publish the crates or document a git/path dependency path until publication is available.
2. Add a short "Using this checkout locally" section with a copy-paste `Cargo.toml` path-dependency example.
3. Add a C API quick start that compiles a tiny C or binding-style program and demonstrates `t4a_last_error_message`.
4. Optionally document that exact C API diagnostic strings are useful for humans but should not be treated as stable programmatic values.
