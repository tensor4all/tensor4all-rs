# N-ary Contraction Pipeline Unification Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Make tensor4all-core contractions express retained shared indices, route dense contraction execution through tenferro-rs N-ary eager einsum, and remove the semantic drift between AD and non-AD paths.

**Architecture:** tensor4all-core owns semantic planning: `DynIndex` matching, `AllowedPairs`, retained output labels, output index order, axis-class propagation, structured payload metadata, and error semantics. tenferro-rs owns dense eager execution through its N-ary einsum APIs. The implementation introduces one contraction plan builder and two executors only where required by storage/AD ownership: an AD executor over `EagerTensor` and a native executor over `tenferro::Tensor`.

**Tech Stack:** Rust, tensor4all-core, tensor4all-tensorbackend, tenferro-rs `eager_einsum_ad` and `eager_einsum_owned`, `cargo test --release`, `cargo fmt --all`, API docs generated by `api-dump`.

/// Options controlling multi-tensor contraction semantics.
///
/// `allowed` determines which tensor pairs may share contraction labels.
/// `retain_indices` marks shared indices that must survive into the output.
#[derive(Clone, Copy, Debug)]
pub struct ContractionOptions<'a> {
    pub allowed: AllowedPairs<'a>,
    pub retain_indices: &'a [DynIndex],
}

impl<'a> ContractionOptions<'a> {
    pub fn new(allowed: AllowedPairs<'a>) -> Self {
        Self {
            allowed,
            retain_indices: &[],
        }
    }

    pub fn with_retain_indices(mut self, retain_indices: &'a [DynIndex]) -> Self {
        self.retain_indices = retain_indices;
        self
    }
}
